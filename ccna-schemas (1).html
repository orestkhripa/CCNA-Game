📶 Modello OSI

Il modello OSI (Open Systems Interconnection) divide la comunicazione di rete in 7 layer indipendenti. Ogni layer ha responsabilità precise e comunica solo con i layer adiacenti.

🏗️
I 7 Layer OSI — Diagramma completo
Con PDU, protocolli e dispositivi di riferimento
DIAGRAMMA
💡 Mnemonico: "All People Seem To Need Data Processing" (dall'alto, L7→L1). Al contrario: "Please Do Not Throw Sausage Pizza Away" (L1→L7).
📦
Esempio: Incapsulamento HTTP
Come un messaggio HTTP diventa bit sul cavo
ESEMPIO REALE
Come leggere lo schema
Ogni header aggiunto contiene info per quel layer. ETH HDR ha MAC src/dst (Layer 2). IP HDR ha IP src/dst (Layer 3). TCP HDR ha porte e numeri di sequenza (Layer 4). Il payload viene mantenuto intatto.

🌐 TCP/IP Stack

Il modello TCP/IP ha 4 layer (o 5 nella versione estesa). È il modello reale usato su Internet, una semplificazione pratica dell'OSI.

Formule Subnetting
FORMULE
Host utilizzabili
2ʰ − 2
h = bit di host (32 − CIDR). Si sottraggono network e broadcast.
Numero di subnet
2ˢ
s = bit di subnet presi in prestito dalla parte host.
Block size (passo)
256 − valore ottetto mask
Es: mask 255.255.255.192 → 256−192=64. Subnet ogni 64.
Broadcast
Network + Block − 1
Es: 192.168.1.64/26 → broadcast = 64+64−1 = 127

🔀 VLANs & Switching

Le VLAN separano logicamente il traffico su uno stesso switch fisico. Essenziale per sicurezza e segmentazione di rete.

Verifica R1# show ip ospf neighbor Neighbor ID Pri State Interface 2.2.2.2 1 FULL/- Fa0/0 R1# show ip route ospf O 10.1.1.0/24 [110/2] via 192.168.1.2, 00:01:23, Fa0/0
💡 La wildcard mask è l'inverso della subnet mask: /24 → 0.0.0.255. OSPF area 0 è la backbone; tutte le altre aree devono connettersi all'area 0.

🔄 NAT & PAT

NAT traduce indirizzi IP privati in pubblici per consentire l'accesso a Internet. PAT (Port Address Translation) è la versione più comune — un solo IP pubblico per tutti.

🛡️ Access Control Lists (ACL)

Le ACL filtrano il traffico su router e switch. Ogni regola viene processata in ordine — la prima che fa match viene applicata.

📋
Standard vs Extended ACL
TABELLA
TIPONUMEROFILTRAPOSIZIONE CONSIGLIATA
Standard1 — 99, 1300-1999Solo IP sorgenteVicino alla destinazione
Extended100 — 199, 2000-2699Src IP, Dst IP, Protocollo, PortaVicino alla sorgente
NamedStringaCome standard/extendedFlessibile, più leggibile
⌨️
Esempi ACL pratici
CLI
STANDARD — blocca rete 10.x
R1(config)# access-list 10 deny 10.0.0.0 0.255.255.255 R1(config)# access-list 10 permit any R1(config)# interface Gi0/1 R1(config-if)# ip access-group 10 in
EXTENDED — permetti solo HTTP
R1(config)# access-list 101 permit tcp 192.168.1.0 0.0.0.255 any eq 80 R1(config)# access-list 101 permit tcp 192.168.1.0 0.0.0.255 any eq 443 R1(config)# access-list 101 deny ip any any ! (implicit deny già esiste, ma esplicito è best practice)
⚠️ Implicit deny: alla fine di ogni ACL c'è sempre un "deny any any" invisibile. Se non inserisci un "permit any", tutto il traffico non matchato viene bloccato!

🌳 Spanning Tree Protocol (STP)

STP previene i loop di rete in topologie con switch ridondanti. Blocca automaticamente le porte ridondanti, lasciandole in standby.

🏆
Elezione Root Bridge & Port States
TABELLA
Elezione Root Bridge
CRITERIODETTAGLIO
1. Bridge PriorityDefault 32768, vince il minore
2. MAC AddressSe priority uguale, vince MAC minore
Stati delle porte STP
STATOAZIONE
BlockingRiceve BPDU, non forwarda
ListeningProcessa BPDU (15s)
LearningImpara MAC table (15s)
ForwardingForwarda il traffico ✓
💡 RSTP (802.1w) è la versione moderna di STP — convergenza in secondi invece di ~50s. PVST+ è la variante Cisco con istanza STP separata per ogni VLAN.

6️⃣ IPv6

IPv6 usa indirizzi a 128 bit (vs 32 bit IPv4). Niente più NAT, autoconfigurazione SLAAC, e molto più spazio di indirizzamento.

📏
Struttura indirizzo IPv6
DIAGRAMMA
Regole di abbreviazione
1. Rimuovi gli zeri iniziali in ogni gruppo: 0db8 → db8, 0000 → 0
2. Sostituisci uno o più gruppi consecutivi di soli zeri con :: (solo una volta)
Esempio: 2001:0db8:0000:0000:0000:0000:0000:0001 → 2001:db8::1
